# Non-Goals

Vehicle Graph is intentionally constrained. The following are not current goals:

- building an AI product, chatbot, or autonomous repair authority;
- replacing factory publications or presenting derived material as factory
  documentation;
- creating a universal automotive ontology before working examples exist;
- storing canonical knowledge primarily in a database or hosted service;
- editing factory facts to match a particular vehicle;
- treating unsupported observations, generated text, or inferred conclusions as canonical facts;
- implementing schemas, application code, or extraction systems during the
  documentation phase;
- adding microservices, queues, caches, distributed systems, or deployment
  infrastructure without a demonstrated need;
- supporting every manufacturer, model, document type, or output format in the
  initial milestone;
- producing a polished platform before producing one trustworthy diagram;
- optimizing for hypothetical future use cases instead of demonstrated engineering needs;
- inferring undocumented engineering relationships to make the graph appear more complete.

These boundaries exist so the repository remains a durable engineering knowledge base rather than becoming another software project.
